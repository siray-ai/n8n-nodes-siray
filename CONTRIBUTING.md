# Contributing to n8n-nodes-siray

Thanks for your interest in contributing. This project is intentionally built as a sequence of small, issue-sized tasks. Please start by opening an issue to discuss what you want to add.

## How to contribute

1. Open an issue describing the change you want to make.
2. Wait for confirmation that it fits the roadmap and direction.
3. Create a branch from `main`.
4. Make your changes with clear commits.
5. Open a pull request that references the issue.

## Development notes

- Keep changes focused and small.
- Prefer incremental additions to existing nodes rather than large refactors.
- Update `README.md` if behavior, setup, or usage changes.
- Add tests when introducing new logic.

## Pull request checklist

- I have opened an issue and linked it in the PR.
- I have tested my changes locally.
- I have updated documentation where needed.
- I have added tests if applicable.
